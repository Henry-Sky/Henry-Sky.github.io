远程桌面(VNC)

2023-04-12 树莓派 407 字
ssh

准备ssh服务

sudo apt install openssh-server
sudo systemctl status ssh.service 
# 确认是否开机自启动
sudo systemctl is-enabled ssh.service

安装服务端

sudo apt install tigervnc-standalone-server
#安装vncsever

基础设置

vncserver
#启动vncsever
#过程中会让你设置密码和只读密码, 自动分配端口号.

启动后

#查看启动后的实例
$ vncserver -list

TigerVNC server sessions:

X DISPLAY #	RFB PORT #	RFB UNIX PATH	PROCESS ID #	SERVER
2         	5902      	             	11368       	Xtigervnc

#如果-list看不到实例, 说明启动失败, 需要journalctl -fe看一下具体原因

其他操作

vncpasswd
#修改密码

vncserver -localhost no
上面启动的服务, 默认是只监听本地127.0.0.1, 其它机器无法连接, 若需要连接, 可以用上述命令启动

配置和添加系统服务

vncserver -kill :2
关闭刚才的实例 :2 根据自己的实例修改

sudo systemctl start tigervncserver@:2.service
sudo systemctl enable tigervncserver@:2.service
将 tigervncservice 服务添加到启动, :2根据自己的实例修改

服务配置文件在 /lib/systemd/system/tigervncserver@.service
使用 /usr/libexec/tigervncsession-start 这个脚本
脚本调用 /usr/sbin/tigervncsession

需要在~/.vnc/下创建文件config
输入内容：

session=ubuntu
#如果前面安装了xfce, 这里可以用xfce, 否则需要改成ubuntu(Desktop版自带桌面)
geometry=1366x768
#开启时的默认分辨率
securitytypes=vncauth,tlsvnc
#如果要只允许本地连接, 可以加一行 localhost

重启服务后生效

sudo systemctl restart tigervncserver@:2.service
